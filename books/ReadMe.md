### Goroutine

不要用共享内存的方式通信。作为替代，应该以通信作为手段来共享内存。

Go语言不推荐以共享内存区的方式传递数据。作为替代，我们应该优先使用Channel。

Channel主要被用来在多个Goroutine之间传递数据，并且还会保证其过程的同步。
